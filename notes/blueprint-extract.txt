1. Executive Summary
Alpitour World operates at the intersection of tour operating, aviation (Neos), hospitality, and a vast agency network through Welcome Travel Group. With brands including Alpitour, Eden Viaggi, Francorosso, Bravo, and Turisanda, the group serves internal teams, B2B agency partners, and B2C consumers through a multi-brand portfolio.
This blueprint defines how AlpiGPT evolves from an information retrieval assistant into an agentic orchestration layer supporting travel agents across the full sales journey — from initial inquiry through needs capture, personalised recommendation, itinerary creation, proposal preparation, and booking support with next-best-action intelligence.
The orchestration substrate is the Kyndryl Agentic Framework (KAF), which coordinates a specialised agent inventory through MCP toolsets against Alpitour's inventory, CRM, and booking platform services. The result is a single conversational touchpoint that delivers the right solution at the right time, reducing manual effort while increasing conversion and agent productivity.
Strategic objective: Transform AlpiGPT into the B2B concierge orchestration layer for ~80% of agency bookings via EasyBook
Scope: Seven-stage sales journey with ten specialised AI agents coordinated through KAF
Target impact: −50% quote turnaround, +30% agent productivity, +25% proposal acceptance
Phase 1 pilot: Standard package quote with sold-out alternative handling — highest volume, lowest integration risk

1. Inquiry
Business Goal: Capture the customer request immediately and initiate an orchestrated work package with zero friction for the travel agent.
User Actions: Travel agent receives client inquiry; opens AlpiGPT within EasyBook; submits natural-language request describing destination, dates, and traveller profile.
AI Agents Involved: Intent Agent
Data Sources Accessed: EasyBook session context; channel authentication; agency credentials
Inputs: Natural-language inquiry; agency ID; session metadata
Outputs: Classified intent; initial work package; conversation thread ID
Business Value: Instant request acceptance eliminates manual triage; every inquiry enters a traceable orchestration pipeline from the first interaction.
2. Needs Capture
Business Goal: Transform ambiguous requests into structured, actionable requirements before any inventory search begins.
User Actions: Agent responds to guided clarifying questions; confirms dates, party size, budget, brand preference, and special requirements.
AI Agents Involved: Requirements Agent (guided by Intent Agent)
Data Sources Accessed: OrchestrationService clarification API; conversation history; agency booking patterns
Inputs: Partial intent; agent clarifications; constraint signals
Outputs: Structured requirements object (destination, dates, pax, budget, brand, constraints)
Business Value: Reduces rework and failed searches; ensures downstream agents operate on complete, validated requirements.
3. Context &amp; Persona Signals
Business Goal: Enrich the request with agency tier, client history, and preference signals to enable personalised orchestration.
User Actions: Agent optionally identifies repeat client or references past bookings; reviews AI-surfaced context summary.
AI Agents Involved: Context Agent
Data Sources Accessed: CRM (agency tier, commission structure); client profiles; booking history API; loyalty data
Inputs: Structured requirements; agency ID; optional client identifier
Outputs: Enriched context profile (tier, commission, preferences, past destinations, spend patterns)
Business Value: Personalisation without manual CRM lookup; agency-tier pricing and recommendations applied automatically.
4. Search &amp; Recommend
Business Goal: Deliver a ranked shortlist of relevant packages, alternatives, and upsell opportunities aligned to agency and client context.
User Actions: Agent reviews ranked options; requests alternatives for sold-out properties; explores upsell suggestions.
AI Agents Involved: Inventory Agent; Recommendation Agent; Upsell Agent
Data Sources Accessed: Inventory API (packages, hotels, Neos flights, allotments); RAG product catalogues; yield status; CRM purchase history
Inputs: Structured requirements; enriched context; search constraints
Outputs: Ranked option shortlist; availability status; alternative proposals; upsell candidates
Business Value: Cuts manual search time by up to 50%; intelligent alternatives prevent lost sales on sold-out inventory.
5. Itinerary Creation
Business Goal: Compose a customised, multi-segment travel solution that satisfies all captured constraints and client preferences.
User Actions: Agent reviews composed itinerary; requests segment adjustments (hotel changes, transfer additions, excursion inclusions).
AI Agents Involved: Itinerary Agent
Data Sources Accessed: Inventory API; itinerary composition rules; destination content (RAG); transfer and excursion catalogues
Inputs: Selected options; structured requirements; context profile
Outputs: Multi-segment itinerary (hotels, flights, transfers, excursions); constraint validation report
Business Value: Enables complex custom trips without call-centre escalation; dynamic KAF workflows handle multi-segment composition.
6. Proposal Preparation
Business Goal: Generate an agency-branded, commercially accurate proposal ready for client presentation.
User Actions: Agent reviews priced proposal; verifies commission visibility; approves or requests revision before client sharing.
AI Agents Involved: Pricing Agent; Proposal Agent
Data Sources Accessed: Pricing engine (B2B markup, commission rules); OPA policy guardrails; brand templates; cancellation and insurance policies (RAG)
Inputs: Composed itinerary; agency tier; commercial rules; brand guidelines
Outputs: Structured quote/proposal (PDF or in-app); commission breakdown; compliance annotations
Business Value: Proposal assembly automated from hours to minutes; consistent commercial accuracy and brand compliance.
7. Booking Support &amp; Next Best Actions
Business Goal: Complete the sale with booking holds or confirmations, contextual upsells, and proactive follow-up actions.
User Actions: Agent confirms booking or places hold; selects upsell options; receives confirmation reference and next-best-action recommendations.
AI Agents Involved: Booking Agent; Upsell Agent
Data Sources Accessed: Booking Platform APIs (EasyBook); hold/reservation services; notification services; CRM for follow-up triggers
Inputs: Approved proposal; agent confirmation; upsell selections
Outputs: Booking reference or hold confirmation; upsell attachments; client notification; next-best-action alerts
Business Value: Accelerates conversion at point of decision; contextual upsells increase average order value; automated follow-up reduces drop-off.
